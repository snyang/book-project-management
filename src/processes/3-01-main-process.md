# 项目管理 - 主流程

## 项目管理要素

- 团队角色和项目角色  
  团队角色是对团队成员在项目管理上的一个逻辑视图。  
  项目角色是对团队成员在软件开发上的一个逻辑视图。  

- 流程  
  规定多个角色如何协作，完成某个产出物。  

- 产出物  
  产出物是角色通过一个流程产生的结果。  

## 团队角色

- 开发者(Developer)  
- 首席开发者(Chief Developer)  
- 项目助理(Project Assistant)  
- 涉众(Stakeholder)  

## 项目角色

- 需求分析师 - Business Analyst  
  - 与涉众交流  
  - 管理涉众的需求，和涉众提供的资料  
  - 编写需求分析  
- 设计工程师 - Design Engineer  
  - 编写设计文档  
- 用户界面设计师 - UI Design Engineer  
  - 设计交互材料  
    比如：Logo、图标、图像、视频、声音  
  - 设计交互界面的风格  
- 架构师 - Architect  
  - 领导相关人员  
  - 管理开发环境、工具、组件的信息  
  - 进行架构设计  
  - 编写架构代码  
- 开发工程师 - Development Engineer  
  - 编码  
  - 编写 API 文档注释  
  - 编写示例代码  
- 测试工程师 - Quality Engineer  
  - 编写测试用例  
  - 测试  
- 运维工程师 - DevOps Engineer  
  - 编写构建脚本  
  - 编写安装程序  
  - 编写运维脚本  
  - 维护运维环境  
  - 定期运行运维程序  
  - 监视每日运维状态  
  - 负责产品发布  
- 信息工程师 - Information Engineer  
  - 编写产品的文档、使用手册  
  - 辅助修改 API 文档  
- 翻译 - Translator  
  - 进行翻译工作  

## 主流程

所有者： PMO  

需求分析师 -> 需求分析书  
需求分析书 -> 设计工程师 -> 设计文档  
设计文档 -> 用户界面设计师 -> UI 设计文档 + UI 设计  
设计文档 + UI 设计文档 -> 开发工程师 -> 代码  
设计文档 -> 运维工程师 -> 运维代码  
代码 + 运维代码 -> 产品  
设计文档 -> 测试工程师 -> 测试用例  
测试用例 -> 测试工程师 -> 测试代码  
测试用例 + 产品 -> 测试工程师 -> 测试结果  
代码 + 测试代码 + 运维代码 -> 测试结果 + 构建结果
